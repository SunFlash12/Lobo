Start here

Welcome to the pack. Six steps and you're on air.

  1. 1
    Change the default password
    Right now the dashboard password is Bladestrex. Copy .env.example to .env, edit DASHBOARD_PASSWORD=…, then restart npm start.
  2. 2
    Connect to your TikTok
    Head to , type your username (no @), hit Connect. Status pill top-right turns green when you go live.
  3. 3
    Upload sounds & images
    — mp3/ogg/wav, png/jpg/gif/webp/svg, or mp4/webm video (25 MB max). Short sounds land harder than long ones.
  4. 4
    Configure the alerts
    Open . For each type, pick a sound + image, tweak the text template, and hit the per-alert Test button. The right-hand preview updates live.
  5. 5
    Add the overlays to OBS
    Go to , copy each overlay URL, and in OBS: + → Browser → paste. All five overlays are transparent — drop them over gameplay.
  6. 6
    Test it all with demo mode
    Before your first stream, open and click Start demo. Random fake events fire every 3-8 seconds so you can watch every overlay come alive without going live.
The stream-day workflow
  1. Boot Lobo Live Lab (npm start)
  2. Open dashboard → Connect
  3. Reset session counters (optional)
  4. Start streaming — status pill turns green — done
Something not working?
Full tutorial with troubleshooting lives in TUTORIAL.md in your project folder. Or check the Command Prompt window running npm start — errors show up there first.

If TikTok blocks your connection while you're clearly live, grab a free Euler Stream key and paste it into .env as SIGN_API_KEY=….

Add to OBS & TikTok LIVE Studio

Overlay URLs

In OBS: + → Browser → paste the URL. Set width/height to match your canvas (1080×1920 portrait works everywhere; 1920×1080 for landscape).

Tip: tick “Shutdown source when not visible” off, and “Refresh browser when scene becomes active” on, so alerts don't miss when you switch scenes.
